# 蓝域出海 BlueReach — 生产部署指南

## 架构

```
用户 → Cloudflare (CDN / WAF / DDoS)
         │
         ├─ shop.your-domain.com → Cloudflare Pages (前端静态)
         └─ api.your-domain.com  → VPS (后端 API)
```

前端免费托管在 Cloudflare Pages，后端需要一台 VPS。

## 第二步：部署后端到 VPS

SSH 登录后：

```bash
# 1. 基础环境
apt update && apt install -y nginx certbot python3-certbot-nginx nodejs npm

# 2. 创建目录
mkdir -p /var/www/bluereach
cd /var/www/bluereach

# 3. 上传后端代码
# 把 server-src/ 整个目录上传到 /var/www/bluereach/server/
# 可以用 scp 或者 git clone

# 4. 安装依赖
cd server
npm install

# 5. 配置环境变量
cp .env.example .env
nano .env
```

**.env 生产配置**：
```bash
NODE_ENV=production
PORT=3000
DATABASE_URL="file:./prod.db"   # SQLite，单机够用
CORS_ORIGINS=https://shop.your-domain.com

# 下面三个必须改！！！
# 生成方式: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
JWT_ACCESS_SECRET=<生成64位hex>
JWT_REFRESH_SECRET=<生成64位hex>
CARD_ENCRYPTION_KEY=<生成64位hex>   # 设定后绝对不能改！

# USDT
TRONGRID_API_KEY=你的key
MERCHANT_WALLET_ADDRESS=你的TRC20地址
```

```bash
# 6. 初始化数据库
npx prisma generate
npx prisma db push
node prisma/seed.js

# 7. 启动
npm i -g pm2
pm2 start src/app.js --name bluereach -i 2
pm2 save && pm2 startup
```

## 第三步：配置 Nginx

```bash
nano /etc/nginx/sites-available/bluereach
```

```nginx
server {
    listen 80;
    server_name api.your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}
```

```bash
ln -s /etc/nginx/sites-available/bluereach /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
certbot --nginx -d api.your-domain.com
```

## 第四步：部署前端到 Cloudflare Pages

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. Workers & Pages → Create → Pages → Upload assets
3. 上传 `client-dist/` 整个目录
4. 绑定域名 `shop.your-domain.com`

或者用命令行：
```bash
npx wrangler pages deploy client-dist --project-name=bluereach
```

## 第五步：Cloudflare 安全配置

### DNS（橙色云朵 = 代理模式）
```
shop.your-domain.com → CNAME → bluereach.pages.dev  (橙色)
api.your-domain.com  → A     → VPS IP               (橙色)
```

### WAF 规则（免费可用 5 条）
```
1. Rate Limiting: /api/admin/login → 5 req/min → Block
2. Block: URI contains /.env or /.git or /wp-admin
3. Block: User-Agent contains "sqlmap" or "nikto"
4. Managed Challenge: Country = (除你的目标市场外)
```

### SSL/TLS
设为 **Full (strict)**

## 安全分析

**已有的防护**：
- AES-256 卡密加密 → 拖库也读不了卡密
- bcrypt 管理员密码 → 拖库也解不出密码
- JWT 双 token → 会话安全
- 防爆破 5次锁15分钟 → 暴力破解无效
- 四层限流 → 拒绝 CC 攻击

**Cloudflare 加持后**：
- 隐藏源站 IP → 攻击者找不到 VPS
- WAF 拦截恶意请求 → SQL注入/XSS 攻击无效  
- DDoS 防护 → 流量攻击被 Cloudflare 吸收
- 免费 CDN → 全球访问加速

**结论**：这套组合对于发卡平台来说安全性足够。真正的风险不在技术层面，在于**供应链**（上游跑路）和**支付**（USDT 收款地址安全）。
